# JobHunter AI — Tailor Resume

## Philosophy

Tailoring a resume is not decoration — it is strategic reframing of real accomplishments
to serve a specific reader (a hiring manager with 7 seconds). Every word on the
tailored resume must come from RESUME.md or the job description — never fabricated.

**Mission:** A person's ability to get a job should be based on their experiences and
capabilities, not on their resume writing skills.

Two-pass approach (borrowed from [gstack/review](https://github.com/garrytan/gstack)):
read everything first, analyze second, write last. Never write a line of the resume
before you've read RESUME.md, the full job description, AND profile.yaml. Skimming leads
to generic resumes that look like every other application.

Evidence before output: show your analysis (what matched, what gaps exist, what you
emphasized) before presenting the HTML. The user should understand *why* each decision
was made, not just receive a document.

Frame skill gaps as investment advice, not failure (borrowed from
[gstack/retro](https://github.com/garrytan/gstack)): "You don't have dbt — worth noting
in a cover letter that you've worked with similar orchestration tools" is better than
"missing: dbt".

**Impact over inventory.** A resume is not a list of tools used or tasks performed.
It is evidence of strategic thinking, business impact, and the ability to deliver
outcomes. Every bullet should answer: "So what? Why did this matter?"

---

## NEVER:
- Write RESUME.md without first showing the user a formatted preview and getting explicit confirmation
- Fabricate experience, skills, or companies — RESUME.md is the only source of truth
- Write the resume before reading RESUME.md, profile.yaml, AND the full job description
- Save a resume file without telling the user exactly where it was saved
- Present the HTML without first showing your analysis (what you changed and why)
- Add skills to the skills section that aren't in RESUME.md
- Invent a summary — base it on actual experience from RESUME.md, angled toward this role
- Stop and tell the user to "follow setup steps" or "run X first" — fix it for them
- Reorder roles out of reverse-chronological order — tailor via bullet selection, not role reordering
- Use AI-detectable filler phrases (see Writing Quality section below)

---

## Writing Quality — Anti-Slop Principles

The resume must read like it was written by a thoughtful human, not generated by AI.

**Avoid these patterns:**
- Generic enthusiasm: "eager to", "passionate about", "excited to join"
- Empty authority claims: "proven track record", "results-driven", "seasoned professional"
- Buzzword chains: "leveraging cutting-edge solutions to drive innovation"
- Filler qualifiers: "various", "numerous", "multiple stakeholders"

**Instead, be specific and concrete:**
- Replace "passionate data professional" → "Built 3 data pipelines processing 200K records/day"
- Replace "proven track record of leadership" → "Managed a 5-person team through a zero-downtime migration"
- Replace "eager to bring expertise to [Company]" → "Built [specific system] similar to [Company]'s [specific product]"

**The test:** Read each sentence aloud. If it could appear on any resume for any job at any company, it's too generic. Rewrite it with specifics from RESUME.md.

## Step 1 — Identify the Job

**By job ID** (from dashboard or `/classify-jobs`):

```bash
python3 -c "
import json, hashlib, pathlib
today = __import__('datetime').date.today().isoformat()
jobs = json.loads(pathlib.Path(f'data/scored/{today}.json').read_text())
jid = 'REPLACE_WITH_ID'   # e.g. a3f9c1d2
match = next((j for j in jobs if hashlib.sha256(j.get('url','').encode()).hexdigest()[:8] == jid), None)
if match:
    print(f\"Title: {match['title']}\")
    print(f\"Company: {match['company']}\")
    print(f\"URL: {match.get('url','')}\")
    print(f\"Score: {match.get('_score', 0):.1f}\")
else:
    print('Job not found — check the ID or try a different date')
    print('Available: ls data/scored/')
"
```

**By company/title search**:

```bash
python3 -c "
import json, hashlib, pathlib
today = __import__('datetime').date.today().isoformat()
jobs = json.loads(pathlib.Path(f'data/scored/{today}.json').read_text())
query = 'netflix'   # adjust
hits = [j for j in jobs if query in j.get('company','').lower() or query in j.get('title','').lower()]
for j in hits[:5]:
    jid = hashlib.sha256(j.get('url','').encode()).hexdigest()[:8]
    print(f'#{jid} — {j[\"title\"]} @ {j[\"company\"]} ({j.get(\"_score\",0):.1f})')
"
```

## Step 2 — Get the Full Job Description

**Using gstack `/browse` (preferred):**

**First, check if gstack is available:**
```bash
BROWSE_OUTPUT=$(~/.claude/skills/gstack/browse/bin/find-browse 2>/dev/null)
B=$(echo "$BROWSE_OUTPUT" | head -1)
if [ -z "$B" ]; then echo "BROWSE_NOT_AVAILABLE"; fi
```

**If gstack is available — fetch the job description:**
```bash
$B goto <job_url>
$B text
```

This works for ANY job URL — including companies not in JBA (Scotiabank, government sites,
LinkedIn listings, etc.). The job does not need to be in the scored dataset.
Validate the output contains job description content (title, requirements, responsibilities).
If the page returned an error or login wall, tell the user and ask them to paste the JD instead.

**After fetching, score and save the browsed job so it enters the pipeline:**
```bash
python3 -c "
from src.matcher import score_and_save_browsed
result = score_and_save_browsed({
    'title': 'REPLACE_TITLE',
    'company': 'REPLACE_COMPANY',
    'url': 'REPLACE_URL',
    'location': 'REPLACE_LOCATION',
    'skill_level': 'REPLACE_LEVEL',
})
print(f\"✅ Scored: {result['_score']:.1f} ({result['_priority']})\")
"
```
This ensures the job appears in the dashboard and `/classify-jobs` even if the user
only came through `/tailor-resume`.

**If gstack is NOT available:** Use AskUserQuestion:
1. **"Install gstack now"** — Show:
   `git clone https://github.com/garrytan/gstack.git ~/.claude/skills/gstack && cd ~/.claude/skills/gstack && ./setup`
   After installing, retry the browse.
2. **"I'll paste the job description"** — Accept pasted text and continue.

**For Greenhouse API jobs** (Stripe, Anthropic, etc.), you can also fetch structured data:
```bash
python3 -c "
import urllib.request, json, re, html
url = 'https://boards-api.greenhouse.io/v1/boards/<company>/jobs/<job_id>'
data = json.loads(urllib.request.urlopen(url, timeout=15).read())
text = html.unescape(data.get('content',''))
text = re.sub(r'<[^>]+>', '\n', text)
print(re.sub(r'\n{3,}', '\n\n', text).strip())
"
```

**If gstack is unavailable and the user didn't install it above:** Ask the user to paste the job description directly.

**Read the ENTIRE description.** Do not skim. Every bullet point and requirement section
matters — the keywords in "Nice to Have" are real signals even if not required.

## Step 4 — Two-Pass Analysis

### Pass 1: Match Assessment (what to emphasize)

Answer these before writing anything:

```
Job requires:       [list required skills from JD]
You have:           [exact matches from RESUME.md]
You're missing:     [gaps — be honest]
Profile context:    [location fit, level fit, relocation relevant?]
Best bullets:       [3-5 bullets from RESUME.md that are most relevant to THIS role]
Reframe needed:     [bullets that are task-oriented and need impact reframing]
Summary angle:      [one sentence connecting your background to this company's specific mission/product]
Page budget:        [1 page if <10yr experience, 2 pages max if 10+yr]
```

Show this analysis to the user before generating HTML. They should see your reasoning.

### Pass 2: Resume Generation (what to write)

With the match assessment complete, build the HTML:

- **Summary** — 2-3 sentences: who you are + what you bring + why this company/role specifically.
  Must contain at least one specific metric from RESUME.md.
  Must reference something specific about THIS company (a product, a mission, a technical challenge).
  Do NOT use filler phrases (see Writing Quality section above).

- **Experience** — ALWAYS reverse-chronological order (most recent role first).
  Within each role, pull forward and emphasize the bullets that answer this job's requirements.
  Bullets that don't resonate for this role can move down or be trimmed.
  **Maximum 3-5 bullets per role.** If the source resume has more, select the most relevant.
  For roles >5 years ago, 2-3 bullets max.

- **Bullet Reframing** — Transform task-oriented bullets into impact-oriented bullets:
  ```
  BEFORE (task-focused):
  "Led design and development of backend services and REST APIs using Java and Spring Boot"

  AFTER (impact-focused):
  "Powered 50,000+ merchants' order processing with 99.99% availability through a
   financial infrastructure handling 100K+ daily requests (Java, Spring Boot, DynamoDB)"
  ```
  Formula: [Business impact / outcome] + [Scale / metric] + [Technologies in parentheses]
  The reader should understand WHY the work mattered, not just WHAT was done.

- **Skills** — job-required skills first, grouped by category.
  Only list skills from RESUME.md. If the job requires something you don't have, note it in gaps.
  Simple comma-separated format — no complex layouts.

- **Education** — degrees with institution and year. Keep brief unless education is highly relevant
  to this role (e.g., PhD for a research position).

- **Gaps** — list honestly. Will go into the cover letter offer at the end.

## Step 5 — Generate HTML Resume

Output a complete, single-page, print-ready HTML file. **This template is ATS-optimized:**
uses standard fonts, simple block layout (no flexbox/grid), standard section headers,
and clean semantic HTML that ATS parsers can extract cleanly.

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>[Your Name] — [Role] at [Company]</title>
  <style>
    /* ATS-SAFE RESUME TEMPLATE — no flexbox, no grid, standard fonts */
    *, *::before, *::after { box-sizing: border-box; margin: 0; padding: 0; }

    body {
      font-family: Calibri, Arial, Helvetica, sans-serif;
      font-size: 11pt;
      color: #1a1a1a;
      max-width: 780px;
      margin: 0 auto;
      padding: 0.5in 0.6in;
      line-height: 1.15;
    }

    /* Name — 18pt, prominent but not oversized */
    h1 {
      font-size: 18pt;
      font-weight: 700;
      text-align: center;
      margin-bottom: 2px;
    }

    /* Contact info — centered, simple, in body (NOT in header/footer) */
    .contact {
      text-align: center;
      color: #333;
      font-size: 10pt;
      margin-bottom: 14px;
    }
    .contact a { color: #1a1a1a; text-decoration: none; }

    /* Section headers — standard names, 12pt, clear separator */
    h2 {
      font-size: 12pt;
      font-weight: 700;
      text-transform: uppercase;
      letter-spacing: 0.05em;
      border-bottom: 1px solid #1a1a1a;
      padding-bottom: 2px;
      margin: 14px 0 6px;
    }

    /* Job entries — simple block layout, no flexbox */
    .job-header {
      margin-bottom: 2px;
    }
    .job-title { font-weight: 700; }
    .job-company { font-style: italic; }
    .job-date { float: right; font-size: 10pt; color: #333; }

    /* Bullet points — standard list */
    ul {
      margin: 3px 0 8px 18px;
      padding: 0;
    }
    li {
      margin-bottom: 2px;
      font-size: 10.5pt;
    }

    /* Skills — simple inline, no grid/flexbox */
    .skills-section p {
      margin-bottom: 2px;
      font-size: 10.5pt;
    }
    .skills-label { font-weight: 700; }

    /* Education */
    .edu-entry { margin-bottom: 4px; }

    /* Print optimization */
    @media print {
      body { padding: 0.4in 0.5in; font-size: 10pt; }
      li { font-size: 10pt; }
      .hint { display: none; }
      a { color: #1a1a1a; }
    }
  </style>
</head>
<body>

  <!-- HEADER — name + contact in body, NOT in page header/footer -->
  <h1>[Full Name]</h1>
  <p class="contact">
    [City, State/Province] · <a href="mailto:[email]">[email]</a> · [phone] · <a href="[linkedin_url]">linkedin.com/in/[handle]</a>
  </p>

  <!-- PROFESSIONAL SUMMARY — tailored to THIS role, anchored in RESUME.md facts -->
  <h2>Professional Summary</h2>
  <p>[2-3 sentences. Must include at least one specific metric from RESUME.md. Must reference
  something specific about THIS company/role. No generic filler.]</p>

  <!-- PROFESSIONAL EXPERIENCE — always reverse-chronological -->
  <h2>Professional Experience</h2>

  <div class="job-header">
    <span class="job-date">[MM/YYYY] – Present</span>
    <span class="job-title">[Job Title]</span> — <span class="job-company">[Company], [Location]</span>
  </div>
  <ul>
    <!-- 3-5 bullets max. Impact-first format: [Outcome + metric] + [how] + (technologies) -->
    <li>[Impact bullet — what business outcome did this achieve? Include metric.]</li>
    <li>[Impact bullet — scale, users affected, revenue/cost impact]</li>
    <li>[Impact bullet — most relevant to THIS job's requirements]</li>
  </ul>

  <!-- Repeat for each role, reverse-chronological. 2-3 bullets for roles >5 years ago. -->

  <!-- SKILLS — job-required skills listed first, simple comma format -->
  <h2>Skills</h2>
  <div class="skills-section">
    <p><span class="skills-label">Languages:</span> [Python, Java, SQL, ...]</p>
    <p><span class="skills-label">Frameworks & Tools:</span> [Spring Boot, React, Docker, ...]</p>
    <p><span class="skills-label">Cloud & Infrastructure:</span> [AWS, GCP, Kubernetes, ...]</p>
  </div>

  <!-- EDUCATION -->
  <h2>Education</h2>
  <div class="edu-entry">
    <span class="job-date">[YYYY]</span>
    <span class="job-title">[Degree]</span> — <span class="job-company">[University]</span>
  </div>

  <p class="hint" style="margin-top:24px;padding:8px;background:#f0f7ff;border-radius:4px;font-size:9pt;color:#555;">
    💡 Press <strong>Cmd+P</strong> (Mac) or <strong>Ctrl+P</strong> (Windows) → <em>Save as PDF</em>
  </p>
</body>
</html>
```

### ATS Compatibility Notes (built into the template above)

- ✅ **Fonts:** Calibri, Arial, Helvetica (ATS standard-safe)
- ✅ **Layout:** Simple block layout, no flexbox, no grid, no tables, no columns
- ✅ **Section headers:** Standard names ("Professional Summary", "Professional Experience", "Skills", "Education")
- ✅ **Contact info:** In body text, not in page header/footer
- ✅ **Date format:** MM/YYYY consistently
- ✅ **Bullet points:** Standard `<ul><li>` elements
- ✅ **No images, graphics, charts, or text boxes**
- ✅ **Font sizes:** 18pt name, 12pt headers, 10.5-11pt body
- ✅ **Margins:** 0.5-0.6in (within 0.5-1in range)
- ✅ **Line spacing:** 1.15 (within 1.0-1.15 range)

## Step 6 — Save and Present

```bash
mkdir -p resumes
# filename: YYYY-MM-DD-company-role.html
# e.g.: resumes/2026-03-16-netflix-analytics-engineer.html
```

**File naming for PDF export:** Tell the user to save the PDF as `FirstName_LastName_Resume.pdf`
(ATS-friendly filename convention).

Present in this order:

1. **Where it's saved:** `resumes/2026-03-16-netflix-analytics-engineer.html`
2. **How to export:** "Open in browser → Cmd+P → Save as PDF → Save as `FirstName_LastName_Resume.pdf`"
3. **What changed** (specific, anchored):
   - "Reframed [Company X] bullet to lead with business impact (50K merchants) instead of technology stack"
   - "Pulled forward the DynamoDB event-driven bullet — they mention exactly-once delivery in their requirements"
   - "Summary references [Company]'s [specific product/mission] based on the JD"
4. **Skill gaps** (as investment advice, not failure):
   - "dbt isn't in your resume — worth a sentence in your cover letter: 'I've built similar pipelines with [X], picking up dbt is on my roadmap'"
5. **One offer:** "Want a cover letter for this role?" (one offer, not a menu)
